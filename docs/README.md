# LLamaVoz — Design documentation

Documents are in the language they were authored in (mostly Spanish; measured results are bilingual-friendly tables).

| Document | What it contains |
|---|---|
| [PRD.md](PRD.md) | Full product requirements document (v0.9, Spanish): personas, 41 functional requirements, NFRs, Windows-specific constraints, tiered insertion strategy, privacy/threat model, compatibility matrix, risks, launch plan. Written under the provisional name **"VozLibre"** — the product was later renamed **LLamaVoz**. |
| [IMPLEMENTATION_PLAN.md](IMPLEMENTATION_PLAN.md) | Concrete engineering plan derived from the PRD: tech stack decisions, solution structure, gated milestones M0–M8, front-loaded risks. |
| [POC-RESULTS.md](POC-RESULTS.md) | Measured proof-of-concept results: insertion tier reliability (POC-1) and local ASR latency/quality (POC-2) — the data behind the streaming architecture decision. |
| [EVAL-RESULTS.md](EVAL-RESULTS.md) | Pipeline evaluation measurements used to calibrate the draft→verify confidence thresholds. |

Current automated test evidence lives at the repo root: [../EVALS-REPORT.md](../EVALS-REPORT.md), regenerated by `RunEvals.cmd`.
